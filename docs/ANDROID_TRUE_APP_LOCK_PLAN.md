# Android Native Pin Genie Layer

Flutter can build the interface, PIN setup, PIN Genie authentication, settings, and app management UI. Android-wide app locking still needs native Android code because unrelated app launches cannot be intercepted by a normal Flutter widget.

This project now includes a native patch script:

```text
tool/patch_android_true_app_lock.py
```

The script patches a generated Flutter Android project with the native pieces below.

## Implemented native pieces

1. **Accessibility service**
   - File generated by patch script:
     ```text
     android/app/src/main/kotlin/com/siam/pingenie/AppLockAccessibilityService.kt
     ```
   - Watches Android window-state changes after the user explicitly enables the service.
   - Checks whether the foreground package is in the Flutter-saved locked package list.
   - Starts the native lock screen when a locked package opens.

2. **Native PIN Genie lock activity**
   - File generated by patch script:
     ```text
     android/app/src/main/kotlin/com/siam/pingenie/NativeLockActivity.kt
     ```
   - Reads the same salted SHA-256 PIN data that Flutter stores through `shared_preferences`.
   - Uses randomized PIN Genie buckets.
   - Temporarily grants unlock access to the target package after the correct PIN is entered.

3. **Flutter-to-Android bridge**
   - File generated by patch script:
     ```text
     android/app/src/main/kotlin/com/siam/pingenie/MainActivity.kt
     ```
   - Method channel:
     ```text
     pin_genie/native_app_lock
     ```
   - Supports checking service status and opening Android Accessibility settings.

4. **Manifest and resource registration**
   - Registers the accessibility service.
   - Registers the native lock activity.
   - Adds package visibility permission for launchable app discovery.

## Required user setup

A user must manually enable the service:

1. Open the app.
2. Create a PIN.
3. Go to **Settings**.
4. Tap **Enable service**.
5. Enable **PIN Genie App Lock** in Android Accessibility settings.
6. Return and tap **Check status**.
7. Select apps to lock.

Android does not allow this permission to be enabled silently.

## Current limitations

- The implementation is designed for sideloaded/internal Android APK builds.
- Some aggressive OEM battery managers may stop accessibility services; users may need to disable battery optimization manually.
- The lock screen is native Android UI, not a Flutter route, because it must appear even when another app is foreground.
- `QUERY_ALL_PACKAGES` should be reviewed before Play Store publication.

## Security rules used

- Plain PIN is never stored.
- Flutter and native Android share only salted PIN hash, salt, PIN length, lock enable state, and locked package names.
- Accessibility service does not collect text, screen content, contacts, files, or private app data.
- Back from the native lock screen sends the user to the home screen instead of letting the locked app through.
